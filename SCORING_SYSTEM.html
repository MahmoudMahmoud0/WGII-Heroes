Hero Scoring System

This document explains the current scoring logic used to analyze heroes for PvP, rally joiners, offense, and defense.

Scope

The scoring system is based on:

  • [heroes_skills.json](</c:/Users/Mahmoud/Desktop/WGII/heroes screenshots/finals/heroes_skills.json>)
  • [hero_types.json](</c:/Users/Mahmoud/Desktop/WGII/heroes screenshots/finals/hero_types.json>)
  • [analyze_heroes.py](</c:/Users/Mahmoud/Desktop/WGII/heroes screenshots/analyze_heroes.py>)

Core Rules

1. PvP scoring uses only `on_map_skills`.

2. Rally joiner scoring uses only the first `on_map_skill`.

3. Hero comparisons should be made inside the same type:

  • `Shooter`
  • `Ranger`
  • `Brawler`

4. Economy-only effects are excluded from PvP scoring.

Data Source

All numeric scoring is based on the maximum values found in `upgrade_preview`.

That means the system does not use the currently visible in-screenshot level. It uses the highest upgrade value available for each skill effect.

Example:

Damage Rate: 115%/138%/161%/184%/230%

The scoring system uses `230%`.

Description Normalization

Skill descriptions are normalized to match the max values from `upgrade_preview` when replacement is safe.

Safe replacements:

  • one percentage in the description -> replaced with the max preview percentage
  • same number of percentages in description and preview tracks -> replaced left to right

Unsafe cases are left unchanged to avoid corrupting text.

Effects Ignored for PvP

These are treated as non-PvP utility:

  • production
  • gathering
  • research
  • training
  • construction
  • resource cost reduction
  • beast or caravan march speed
  • stamina cost reduction

Proc Chance Handling

If a skill includes chance-based wording, the score is discounted by that chance.

Recognized examples:

  • `has 50% chance`
  • `with a 10% chance`
  • `25% chance`

Example:

Damage Dealt: 200%
With a 25% chance

Effective scored value:

200 x 0.25

Rally Bonus Handling

If an effect is rally-related, it gets a rally multiplier.

Current rally multiplier:

1.25

Skill Score Formula

For one recognized effect:

effect score = max upgrade value x effect weight x proc chance x rally bonus

For one skill:

skill score = sum of all recognized effects inside that skill

Hero PvP Score

The hero PvP score is the total of all on-map skill scores:

pvp_score = sum(all on_map skill scores)

Offense and Defense Split

Each effect also contributes to either offense, defense, or both.

Mostly offensive

  • `enemy troop damage received`
  • `damage dealt`
  • `enemy troop defense weakening`
  • `troop attack enhancement`

Mostly defensive

  • `received damage reduction`
  • `damage dealt by enemy troop reduction`
  • `troop defense enhancement`
  • `troop life enhancement`

Mixed

  • `enemy troop attack weakening`
  • mostly defense
  • slight offense credit

This produces:

  • `offense_score`
  • `defense_score`

Combat Role

Each hero is labeled:

  • `Attack`
  • `Defense`
  • `Hybrid`

Current rule:

Attack  if offense_score > defense_score x 1.2
Defense if defense_score > offense_score x 1.2
Hybrid  otherwise

Rally Joiner Scores

Only the first `on_map_skill` is used.

The system computes:

  • `rally_joiner_score`
  • `rally_offense_score`
  • `rally_defense_score`

Rally Ranking

Attack and defense are not mixed for rally evaluation.

This means:

  • `rally_offense_score` ranks attack-oriented rally joiners
  • `rally_defense_score` ranks defense-oriented rally joiners
  • `rally_joiner_score` remains the raw first-skill total for reference only

Ranking Priorities

PvP ranking

sort by pvp_score
then rally score

Rally attack ranking

sort by rally_offense_score
then rally_joiner_score
then pvp_score

Rally defense ranking

sort by rally_defense_score
then rally_offense_score
then rally_joiner_score
then pvp_score

Attack ranking

sort by offense_score
then pvp_score

Defense ranking

sort by defense_score
then pvp_score

Fallback Behavior

If OCR misses a proper stat line but the description still looks combat-related, a small fallback score may be assigned.

Examples:

  • `damage dealt` text -> small offense fallback
  • `damage reduction` text -> small defense fallback
  • `troop attack` text -> small offense fallback
  • `enemy attack` text -> small defense fallback

This helps avoid zero scores caused purely by OCR wording issues.

Examples

Emily

First on-map skill:

  • `Damage Dealt: 20%`
  • `Rally DMG Dealt Enhancement: 5%`

Because `damage dealt` is weighted higher than plain `troop attack`, Emily scores strongly as an offensive rally joiner.

Red Cloud

First on-map skill:

  • `Troop Attack: 20%`
  • `Rally Troop Attack Enhancement: 5%`

This is still good, but slightly lower than Emily because the current system values `damage dealt` above `troop attack`.

Eleanor Dumont

Her kit is chance-based, so her effects are discounted by proc chance.

Examples:

  • `250%` at `10% proc`
  • `200%` at `25% proc`
  • `60%` reduction at `40% proc`

This prevents chance-based heroes from being overvalued as if their effects were always active.

Practical Reading Guide

Use the scores like this:

  • `pvp_score` tells you overall on-map combat value
  • `offense_score` tells you pressure and damage value
  • `defense_score` tells you survivability and mitigation value
  • `rally_offense_score` tells you who is strongest for attack-oriented rally joining
  • `rally_defense_score` tells you who is strongest for defensive rally joining

Best practice:

  • compare heroes inside the same type
  • compare attack heroes with attack heroes
  • compare defense heroes with defense heroes
  • use mixed-type rankings only as rough overviews

Notes

This is a heuristic text-based scoring system, not a full combat simulator.

It does not model:

  • hidden game formulas
  • exact troop stat scaling
  • timing interactions
  • skill sequencing
  • undocumented in-game modifiers

It is meant to provide a structured, editable comparison model from the extracted skill text.
